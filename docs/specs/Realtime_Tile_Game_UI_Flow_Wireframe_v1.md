## 1. 화면 설계 목표

이 프로젝트의 화면은 화려한 연출보다 다음을 우선한다.

- 현재 상태를 한눈에 이해할 수 있어야 한다.
- 상대 손패 내용은 노출하지 않고 남은 개수만 보여야 한다.
- 현재 턴과 남은 시간이 명확해야 한다.
- 손패 자동 정렬과 자유 정렬을 모두 지원해야 한다.
- 잘못 배치하거나 시간이 초과되면 배치 전 상태로 돌아간다는 점을 보여야 한다.
- 네트워크 지연이나 중복 클릭 상황에서 사용자가 같은 행동을 반복하지 않게 해야 한다.
- 2인, 3인, 4인 모두 같은 화면 구조에서 자연스럽게 표현되어야 한다.

## 2. 전체 사용자 흐름

```text
랜딩
→ 회원가입 또는 로그인
→ 프로필 설정
→ 로비
→ 방 생성 또는 방 입장
→ 대기방
→ 전원 준비
→ 방장 게임 시작
→ 게임 화면
→ 승리 또는 교착 종료
→ 결과 화면
→ 재경기 또는 로비 복귀
```

재접속 흐름:

```text
게임 중 새로고침 또는 연결 끊김
→ 로그인 상태 복구
→ 진행 중 게임 감지
→ 재접속 안내
→ 기존 방과 게임 화면 복원
```

# 4. 랜딩 화면

## 목적

서비스 소개와 로그인·회원가입 진입점을 제공한다.

## 주요 요소

```text
[자체 게임 로고]

실시간 타일 보드게임
2~4명이 함께 즐기는 숫자 타일 전략 게임

[로그인]
[회원가입]
```

## 와이어프레임

```text
┌──────────────────────────────────────────────────────┐
│                      GAME LOGO                       │
│                                                      │
│            실시간 타일 보드게임                       │
│     2~4명이 함께 즐기는 숫자 타일 전략 게임           │
│                                                      │
│             [ 로그인 ]   [ 회원가입 ]                 │
│                                                      │
│        그룹 · 연속수 · 조커 · 실시간 대전             │
└──────────────────────────────────────────────────────┘
```

## 정책

- 이미 로그인한 사용자는 `/lobby`로 이동한다.
- 상용 루미큐브 로고나 이미지 리소스를 사용하지 않는다.

- 새로고침
- 방 생성

- 방 이름

- 현재 인원 / 최대 인원
- 상태
- 비밀번호 여부

1차 MVP에서는 비밀번호 방을 제외할 수 있으나 UI 공간은 확장 가능하게 둔다.

# 10. 대기방 화면

## 목적

참가자 확인, 준비 상태 변경, 방장 시작을 처리한다.

## 주요 요소

- 방 이름
- 방 코드 또는 방 ID
- 참가자 카드
- 준비 버튼
- 방 나가기
- 방장 시작
- 최대 인원
- 게임 규칙 요약

## 와이어프레임

```text
┌──────────────────────────────────────────────────────────────┐
│ ← 로비          초보방                     인원 3/4           │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│  ┌─────────────┐ ┌─────────────┐ ┌─────────────┐ ┌─────────┐│
│  │ 👑 방장      │ │ 참가자      │ │ 참가자      │ │ 빈 자리 ││
│  │ player1     │ │ player2     │ │ player3     │ │         ││
│  │ 준비 완료   │ │ 준비 완료   │ │ 준비 전     │ │         ││
│  └─────────────┘ └─────────────┘ └─────────────┘ └─────────┘│
│                                                              │
│  규칙: 첫 등록 30점 · 턴 제한 120초 · 2~4인                 │
│                                                              │
│     [방 나가기]         [준비하기]         [게임 시작]        │
└──────────────────────────────────────────────────────────────┘
```

## 방장 정책

- 최초 방 생성자가 방장
- 방장 이탈 시 입장 순서가 가장 빠른 사용자에게 위임
- 방장도 준비 상태가 필요함
- 최소 2명이 있어야 시작 가능
- 모든 참가자가 준비 완료여야 시작 가능
- 최대 인원을 채우지 않아도 최소 2명 이상이면 시작 가능

## 시작 실패 메시지

```text
최소 2명의 참가자가 필요합니다.
모든 참가자가 준비를 완료해야 합니다.
이미 게임이 시작되었습니다.
```

## 11-1. 상대 플레이어 표시

각 플레이어 카드에 표시:

- 아바타
- 닉네임
- 방장 여부
- 손패 개수
- 첫 등록 완료 여부
- 현재 턴 여부
- 연결 상태

표시 예:

```text
player2
손패 8개
첫 등록 완료
현재 턴
● 연결됨
```

절대 포함하지 않는 정보:

- 상대 tileId
- 상대 타일 숫자
- 상대 타일 색상
- 상대 손패 정렬 순서

## 11-5. 자동 정렬 버튼

### `777`

같은 숫자를 우선으로 모은다.

정렬 우선순위:

```text
number ASC
→ color order
→ duplicate copy order
```

권장 색상 순서:

```text
RED → BLUE → YELLOW → BLACK
```

예:

```text
R7, B7, Y7, K7 | R8, B8 | R10
```

### `789`

같은 색상 안에서 숫자 오름차순으로 정렬한다.

정렬 우선순위:

```text
color order
→ number ASC
→ duplicate copy order
```

예:

```text
RED 1, RED 2, RED 3, RED 8
BLUE 4, BLUE 5, BLUE 6
BLACK 11, BLACK 12
```

연속 후보 사이에 시각적 간격을 줄 수 있다.

### 조커 위치

자동 정렬 시 조커는 오른쪽 끝 별도 영역에 둔다.

```text
일반 타일 ... | JOKER | JOKER
```

조커를 특정 RUN이나 GROUP에 자동 삽입하지 않는다.

### 정렬 상태 저장

1차 정책:

- 정렬은 클라이언트 표시 순서만 변경
- 서버 게임 상태와 무관
- WebSocket 요청 없음
- 다른 사용자에게 전송하지 않음
- 브라우저 새로고침 시 기본 정렬로 복원 가능
- 추후 사용자 설정으로 마지막 정렬 모드 저장 가능

### `원래 순서`

턴 시작 또는 마지막 서버 수신 순서로 손패를 복원한다.

사용자가 수동으로 만든 순서를 보존하려면 별도의 `자유 정렬` 상태를 둔다.

---

## 11-6. 행동 버튼

### 턴 확정

활성 조건:

- 자신의 턴
- 서버 요청 진행 중 아님
- 임시 변경 존재
- 클라이언트 기본 검증 통과

클릭 후:

```text
버튼 비활성화
→ 서버 검증
→ 성공 시 새 GameState 수신
→ 실패 시 오류 표시
```

### 턴 취소

```text
현재 TurnDraft 폐기
→ 턴 시작 스냅샷 상태 표시
→ 턴은 유지
```

### 타일 뽑기

클릭 시 확인:

```text
배치 중인 타일이 있습니다.
현재 배치를 취소하고 타일을 뽑을까요?
```

확인 후:

```text
배치 전 상태 복원
→ 서버 드로우 요청
→ 타일 1개 수신
→ 다음 턴
```

### PASS

타일 풀이 비어 있을 때만 표시한다.

## 11-7. 턴 타이머

표시:

```text
01:24
```

경고 단계:

```text
30초 이하
→ 경고 표시

10초 이하
→ 강한 경고 및 짧은 효과음 선택 가능
```

시간 초과:

```text
서버 타이머 만료
→ 현재 TurnDraft 폐기
→ 턴 시작 스냅샷 복원
→ 손패·테이블·조커 위치 모두 배치 전으로 복원
→ 타일 풀이 남았으면 1개 자동 드로우
→ 풀이 비었으면 PASS
→ 다음 플레이어
```

화면 메시지:

```text
제한시간이 종료되어 배치 전 상태로 복원되었습니다.
타일 1개를 자동으로 뽑고 턴이 종료됩니다.
```

중요:

- 클라이언트 타이머는 표시용
- 실제 시간 초과 판정은 서버 시간 기준
- 클라이언트가 0초를 표시했다고 임의로 상태를 변경하지 않음
- 시간 초과 결과는 서버 이벤트를 받아 반영

## 11-8. 첫 등록 상태

첫 등록 미완료 플레이어의 화면:

```text
첫 등록이 필요합니다.
자신의 손패만 사용하여 합계 30점 이상의 조합을 만드세요.
```

첫 등록 턴에는 기존 테이블 타일 드래그를 비활성화한다.

서버에서도 재검증한다.

## 11-9. 오류 메시지

예:

```text
현재 차례가 아닙니다.
첫 등록 점수가 30점보다 낮습니다.
첫 등록 전에는 기존 테이블을 변경할 수 없습니다.
테이블에 유효하지 않은 조합이 있습니다.
손패에서 최소 1개 이상의 타일을 사용해야 합니다.
회수한 조커는 같은 턴에 다시 사용해야 합니다.
게임 상태가 변경되었습니다. 최신 상태를 다시 불러옵니다.
```

오류는 코드와 사용자 메시지를 분리한다.

# 19. 접근성 및 사용성

- 색상만으로 타일을 구분하지 않는다.
- 타일에 색상 약어 또는 패턴을 함께 표시한다.
- 현재 턴은 색상, 테두리, 텍스트를 함께 사용한다.
- 버튼에 아이콘만 두지 않고 텍스트를 함께 둔다.
- 오류 메시지는 사라지기 전에 충분한 시간 표시한다.
- 드래그가 어려운 사용자를 위해 클릭 선택 후 이동 방식도 확장 가능하게 설계한다.

색상 약어 예:

```text
R = RED
B = BLUE
Y = YELLOW
K = BLACK
```
