## 3. 포트폴리오에 넣기 좋은 핵심 문제 해결 사례

최소 3개 이상 확보한다.

### 사례 A. 비공개 손패 노출 방지

확인할 내용:

- 서버가 전체 손패 상태를 가지고 있는가
- 각 플레이어에게 자기 손패만 보내는가
- 브라우저 개발자 도구에서 상대 손패가 보이지 않는가
- 관전자나 잘못된 사용자 ID 요청으로 조회할 수 없는가

### 사례 B. 중복 턴 요청과 동시성

확인할 내용:

- 확정 버튼을 연속 클릭했을 때 한 번만 처리되는가
- 네트워크 지연 중 같은 요청이 다시 들어와도 중복 반영되지 않는가
- 현재 턴이 아닌 플레이어 요청이 차단되는가
- 상태 버전이 오래된 클라이언트 요청이 거부되는가

### 사례 C. 잘못된 배치 및 시간 초과 원상복구

확인할 내용:

- 잘못된 조합을 확정할 때 서버 상태가 바뀌지 않는가
- 손패와 테이블이 턴 시작 상태로 돌아가는가
- 조커가 포함된 실패 케이스도 복원되는가
- 잘못 배치한 상태로 시간이 초과되어도 배치 전 상태로 돌아가는가
- 시간 초과 복원 후 자동 드로우 또는 PASS가 정확히 한 번만 처리되는가
- 다른 플레이어 화면도 동일한 복원 결과를 받는가

### 사례 D. 재접속 복원

확인할 내용:

- 새로고침 후 동일 게임으로 돌아오는가
- 자신의 손패만 복원되는가
- 현재 턴과 공개 테이블이 일치하는가
- 여러 번 재접속해도 중복 플레이어가 생기지 않는가

## 9. 추가 포트폴리오 문제 해결 사례

### 사례 E. 다인전 랭킹 계산과 중복 반영 방지

확인할 내용:

- 자신을 제외한 참가자 평균 랭킹을 정확히 계산하는가
- 낮은 상대 평균에게 이기면 획득 점수가 작아지는가
- 높은 상대 평균에게 이기면 획득 점수가 커지는가
- DRAW가 actualScore 0.5로 반영되는가
- SPEED 결과가 랭킹에 반영되지 않는가
- 같은 종료 이벤트 재전송 시 랭킹이 중복 반영되지 않는가

### 사례 F. 특정 상대 전적 집계

확인할 내용:

- 사용자 A의 승리가 사용자 B 관점에서는 패배로 기록되는가
- DRAW가 양쪽 모두 무승부로 기록되는가
- CLASSIC과 SPEED가 구분되는가
- 전체 전적과 모드별 전적이 일치하는가
- 원본 게임 결과와 집계 테이블이 일치하는가
