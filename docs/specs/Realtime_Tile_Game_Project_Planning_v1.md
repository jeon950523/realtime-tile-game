# 실시간 타일 보드게임 프로젝트 기획 확정본 v1

작성 기준: 2026-07-14  
프로젝트 성격: 학원 내부망 기반 2~4인 실시간 멀티플레이 웹 게임  
개발 방식: 생성형 AI 코드 작성 + 사용자 직접 실행·검증·테스트·품질 판단

---

## 1. 프로젝트 한 줄 소개

2~4명의 사용자가 학원 내부망에서 방을 만들고 실시간으로 타일을 배치하며 플레이하는 루미큐브 계열의 웹 보드게임 서비스.

> 상용 브랜드명, 로고, 이미지 리소스를 복제하지 않고 자체 명칭과 자체 제작 타일 UI를 사용하는 것을 원칙으로 한다.

## 7. 권장 서버 구조

```text
Browser
  ├─ REST API
  │   ├─ 회원가입
  │   ├─ 로그인
  │   ├─ 방 목록
  │   └─ 재접속 초기 상태 조회
  │
  └─ WebSocket
      ├─ 방 입장/퇴장
      ├─ 준비 상태
      ├─ 게임 시작
      ├─ 타일 배치
      ├─ 턴 확정/취소
      ├─ 타일 뽑기
      └─ 게임 종료
```

게임 상태의 최종 판단 권한은 서버가 가진다.

```text
클라이언트
→ 사용자의 의도 전달

서버
→ 현재 턴, 소유권, 규칙, 상태 버전 검증
→ 성공 시 게임 상태 변경
→ 각 플레이어에게 필요한 정보만 전송
```

---

## 8. 핵심 보안 및 무결성 원칙

- 클라이언트가 보낸 손패 정보를 신뢰하지 않는다.
- 서버가 타일 소유권을 관리한다.
- 다른 플레이어의 손패 원문을 전송하지 않는다.
- 방과 게임 요청은 인증 사용자 기준으로 처리한다.
- 현재 턴이 아닌 사용자의 행동은 거부한다.
- 동일 요청 중복 실행을 방지한다.
- 유효하지 않은 배치는 게임 상태를 변경하지 않는다.
- 턴 시작 스냅샷을 보관해 취소 또는 실패 시 복원한다.
- 게임 상태에는 버전 번호를 두어 오래된 요청을 감지한다.

전체 상태를 매 행동마다 관계형 테이블에 잘게 저장하는 방식은 1차에서 피한다.

## 17. 게임 모드 및 랭킹 확장

### 게임 모드

```text
CLASSIC
- 첫 등록 30점
- 손패를 모두 사용하면 승리
- 일반 랭킹 점수 반영

SPEED
- 전체 게임 제한시간 5분
- 턴 제한시간 기본 20초
- 첫 등록 30점 미적용
- 제한시간 종료 후 점수 승부
- 랭킹 점수 미반영
```

### 동점

모든 모드에서 최종 승자를 결정할 수 없는 동점은 `DRAW` 처리한다.

### 상대 전적

내 정보 화면에서 특정 상대와의 다음 전적을 확인할 수 있게 한다.

```text
상대 닉네임
총 대전 수
승
패
무
승률
최근 대전일
```

### 랭킹 점수

- 일반모드(CLASSIC)만 랭킹 점수에 반영한다.
- 스피드모드는 친선·짧은 경기 성격으로 랭킹에 반영하지 않는다.
- 상대 평균 점수가 낮으면 승리 보상이 작다.
- 비슷한 상대에게 이기면 보통 수준의 점수를 얻는다.
- 높은 점수의 상대에게 이기면 더 많은 점수를 얻는다.
- 패배 시에는 상대 평균 점수에 따라 차감량이 달라진다.
- DRAW는 기대 승률에 따라 소폭 상승 또는 하락할 수 있다.
